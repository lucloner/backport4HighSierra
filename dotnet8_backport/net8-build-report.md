# .NET 8 Runtime Build Report on macOS 10.13.6

## Environment
- **macOS**: 10.13.6 High Sierra (x86_64)
- **Hardware**: Macmini5,2 (Mac Mini 2012)
- **Xcode**: 10.1 (AppleClang 10.0.0)
- **CMake**: 4.1.0
- **Source**: dotnet/runtime release/8.0 branch

## Key Modifications Required

### 1. macOS Deployment Target
- Changed `CMAKE_OSX_DEPLOYMENT_TARGET` from `10.15` to `10.13` in `eng/native/configurecompiler.cmake`
- Changed `macOSVersionMin` from `10.15` to `10.13` in `Directory.Build.props`

### 2. CryptoKit → OpenSSL Fallback
- Patched `src/native/libs/CMakeLists.txt` to use `System.Security.Cryptography.Native` (OpenSSL) instead of `System.Security.Cryptography.Native.Apple` (CryptoKit)
- CryptoKit requires macOS 10.15+, not available on 10.13

### 3. Swift Compilation Flags
- Removed `-static` and `-runtime-compatibility-version none` flags from Swift compilation in `System.Security.Cryptography.Native.Apple/CMakeLists.txt`
- Swift 4.2.1 (bundled with Xcode 10.1) doesn't support these flags

### 4. AVX-512 Assembly Support
- Added `-mavx512f -mavx512bw -mavx512vl` to `CMAKE_ASM_FLAGS`, `CMAKE_C_FLAGS`, and `CMAKE_CXX_FLAGS`
- Required for compiling `context2.S` which contains AVX-512 instructions

### 5. Const Correctness Fix
- Fixed `char*` → `const char*` in `src/native/corehost/hostmisc/utils.cpp:488`

### 6. Version Stub Files
- Created stub `_version.h`, `_version.c`, and `runtime_version.h` in `artifacts/obj/`
- Normally generated by the managed build (which requires .NET SDK)

### 7. CMake Configuration Parameters
```
-DCMAKE_BUILD_TYPE=Release
-DCMAKE_OSX_ARCHITECTURES=x86_64
-DCMAKE_OSX_DEPLOYMENT_TARGET=10.13
-DCMAKE_OSX_SYSROOT=/Applications/Xcode.app/Contents/Developer/Platforms/MacOSX.platform/Developer/SDKs/MacOSX10.14.sdk
-DCLI_CMAKE_PKG_RID=osx.10.13-x64
-DCLI_CMAKE_FALLBACK_OS=osx
-DCLI_CMAKE_COMMIT_HASH=custom
-DOPENSSL_ROOT_DIR=/usr/local/opt/openssl@3
```

## Build Results

### Successfully Built (592 targets)
- `libcoreclr.dylib` (7.3MB) - CoreCLR runtime engine
- `libclrjit.dylib` - JIT compiler
- `libmscordaccore.dylib` - Debug access
- `libmscordbi.dylib` - Debug interface
- `libclrgc.dylib` / `libclrgcexp.dylib` - GC
- `libcoreclr_static.a` (482MB) - Static library
- Various other native libraries

### Failed Targets
- `singlefilehost` - Missing static libraries for System.Security.Cryptography.Native.Apple (replaced with OpenSSL)
- `debug/di/process.cpp` - Missing version defines (RuntimeProductMajorVersion etc.)
- `debug/ee/rcthread.cpp` - Missing version defines
- `vm/eventtrace.cpp` - Missing version defines

## What Did NOT Work
- **build.sh (managed build)**: Requires .NET 8+ SDK which needs macOS 10.15+
- **.NET 6 SDK**: Works on 10.13 but Arcade SDK tools require .NET 8+
- **.NET 8/10 SDK**: Binaries require macOS 10.15+/12.0+ respectively
- **Docker**: No running daemon available
- **Full native build**: Some targets fail due to missing managed-code-generated files

## Conclusion
Partial success: CoreCLR native runtime components can be built on macOS 10.13.6
with significant patching. A complete build (including managed libraries, host, 
and test infrastructure) requires a macOS 10.15+ environment or a Linux VM/container.
